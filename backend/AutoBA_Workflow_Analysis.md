# AutoBA 系统工作流程详细分析

## 1. 系统启动流程

### 启动程序：
- **main.py**：FastAPI 应用主入口
- **src/server.py**：API 服务器配置
- **src/agent.py**：Agent 核心逻辑
- **src/executor.py**：命令执行器

### 工作内容：
1. 启动 FastAPI 服务器，监听 `0.0.0.0:8004` 端口
2. 配置 CORS 中间件，允许跨域请求
3. 挂载前端静态文件（如果存在）
4. 初始化 API 路由，包括任务创建、查询、控制等接口
5. 启动后台任务管理器

### 调试关键点：
- 检查端口是否被占用
- 确认 Docker 服务是否运行
- 验证前端文件是否正确挂载

## 2. 任务创建流程

### 涉及程序：
- **main.py**：处理 API 请求
- **src/agent.py**：创建 Agent 实例
- **src/data_precheck.py**：数据预检

### 工作内容：
1. 接收用户上传的文件和任务配置
2. 生成唯一的任务 ID（UUID）
3. 创建任务目录结构：
   - `backend/output/task_{task_id[:8]}/data/`：存储输入数据
   - `backend/output/task_{task_id[:8]}/output/`：存储输出结果
4. 保存上传的文件到数据目录
5. 运行数据预检，验证文件格式和完整性
6. 创建 Agent 实例，启动任务执行

### 调试关键点：
- 检查文件上传是否成功
- 验证任务目录是否正确创建
- 确认数据预检是否通过

## 3. 任务执行流程

### 涉及程序：
- **src/agent.py**：Agent 核心逻辑
- **src/prompt.py**：提示词生成
- **src/executor.py**：命令执行
- **src/local_llm.py**：本地模型支持（可选）

### 工作内容：
1. **第一轮（计划生成）**：
   - 调用 `Agent.run_plan_phase()`
   - 生成分析计划提示词
   - 调用远程 API（如 Kimi API）生成分析计划
   - 解析并保存生成的计划

2. **后续轮次（代码执行）**：
   - 循环执行计划中的每个步骤
   - 生成代码生成提示词
   - 调用远程 API 生成执行代码
   - 执行生成的代码
   - 处理执行结果
   - 更新任务状态和日志

### 调试关键点：
- 检查 API 调用是否成功
- 验证生成的计划和代码是否合理
- 确认任务状态更新是否正确

## 4. 模型调用流程

### 涉及程序：
- **src/agent.py**：模型调用逻辑
- **src/prompt.py**：提示词生成
- **src/local_llm.py**：本地模型支持（可选）

### 工作内容：
1. **提示词生成**：
   - 根据任务类型和历史记录生成提示词
   - 包含系统信息、输入数据、历史记录、任务目标等
   - 格式化为 API 要求的格式

2. **API 调用**：
   - 选择模型引擎（如 Kimi API）
   - 构建 API 请求
   - 发送请求并接收响应
   - 处理 API 错误和重试

3. **响应解析**：
   - 解析 API 响应为结构化数据
   - 提取计划或代码
   - 验证响应格式是否正确

### 调试关键点：
- 检查 API 密钥是否正确
- 验证提示词格式是否符合 API 要求
- 确认响应解析是否正确

## 5. 命令执行流程

### 涉及程序：
- **src/executor.py**：命令执行器
- **Docker**：容器化执行环境

### 工作内容：
1. **命令解析**：
   - 识别命令类型（Trimmomatic、SPAdes、Samtools 等）
   - 提取命令参数和文件路径

2. **文件路径处理**：
   - 检查输入文件是否存在
   - 处理各种文件命名模式（R1/R2、cleaned_R1/cleaned_R2 等）
   - 修复文件路径不匹配问题

3. **Docker 容器执行**：
   - 选择合适的 Docker 镜像
   - 检查镜像是否存在，不存在则拉取
   - 构建 Docker 命令
   - 执行命令并捕获输出
   - 处理执行错误

4. **错误处理**：
   - 检测常见错误（文件不存在、格式错误等）
   - 尝试自动修复错误
   - 提供详细的错误信息

### 调试关键点：
- 检查 Docker 服务是否运行
- 验证文件路径转换是否正确
- 确认 Docker 镜像是否存在
- 检查命令执行日志是否详细

## 6. 结果处理流程

### 涉及程序：
- **src/agent.py**：结果处理
- **main.py**：API 响应
- **src/executor.py**：输出处理

### 工作内容：
1. **命令执行结果处理**：
   - 解析命令执行输出
   - 检查执行是否成功
   - 提取关键信息和错误

2. **任务状态更新**：
   - 根据执行结果更新任务状态
   - 记录执行时间和资源使用

3. **输出文件管理**：
   - 收集输出文件
   - 生成文件清单
   - 准备下载链接

4. **API 响应**：
   - 构建任务状态响应
   - 提供输出文件信息
   - 返回执行结果和日志

### 调试关键点：
- 检查任务状态更新是否正确
- 验证输出文件是否正确生成
- 确认 API 响应格式是否符合要求

## 8. 常见问题和解决方案

| 问题 | 原因 | 解决方案 | 调试文件 |
|------|------|----------|----------|
| Docker 不可用 | Docker 服务未运行 | 启动 Docker Desktop | src/executor.py |
| 文件路径错误 | 文件命名模式不匹配 | 检查文件命名规范，使用正确的路径格式 | src/executor.py |
| 命令执行失败 | 工具参数错误 | 检查命令格式和参数，参考工具文档 | src/executor.py |
| API 调用失败 | API 密钥错误或网络问题 | 检查 API 密钥和网络连接 | src/agent.py |
| 任务执行超时 | 分析任务过于复杂 | 优化分析流程，增加超时时间 | src/agent.py |
| 内存不足 | 基因组组装等任务需要大量内存 | 确保系统有足够的内存，调整任务参数 | src/executor.py |

## 9. 调试和修改建议

### 调试技巧：
1. **启用详细日志**：修改 `src/executor.py` 中的日志级别，增加详细信息
2. **检查 Docker 状态**：使用 `docker ps` 和 `docker images` 命令检查 Docker 状态
3. **测试 API 调用**：使用 Postman 或 curl 测试 API 端点
4. **文件路径验证**：在执行命令前验证文件路径是否正确
5. **模拟执行**：在执行实际命令前，先打印命令内容进行验证

### 修改建议：
1. **添加新工具**：
   - 在 `src/executor.py` 的 `docker_images` 字典中添加新工具的镜像
   - 在命令解析逻辑中添加新工具的处理

2. **优化文件路径处理**：
   - 扩展 `src/executor.py` 中的文件命名模式处理
   - 增加更多的路径转换规则

3. **改进错误处理**：
   - 在 `src/executor.py` 中添加更多的错误检测和处理
   - 提供更详细的错误信息和解决方案

4. **增强模型调用**：
   - 在 `src/agent.py` 中添加更多的模型引擎支持
   - 优化提示词生成逻辑

5. **添加监控和统计**：
   - 在 `src/agent.py` 中添加任务执行统计
   - 实现任务进度监控
